# 设计原则 

### S.O.L.I.D

**Single-responsibility Principle**

一个类只负责一个职责



**Open-closed Principle**

添加一个新功能应该是在已有代码基础上扩展代码（新增模块，类，方法），而非修改已有代码



**Liskov Substitution Principle**

子类可以被当作父类来用，且需要比父类更特殊



**The Interface Segregation Principle**

多个单一的接口要要优于包含很多功能的总接口
